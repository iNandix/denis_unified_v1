Perfil CODING
Objetivo: implementar cambios mínimos, testeables, sin stubs.
DoD: ruff/pytest si existen + smoke de fase afectada + artifact JSON.
Disciplina: timeouts en red/SSE, fallback, logs de diagnóstico.
Antes de escribir código nuevo “grande”:
1) Buscar en GH/HF con palabras clave (componente + tecnología).
2) Evaluar 2–3 opciones avanzadas.
3) Integrar la mejor opción o, si no encaja, explicar por qué se descarta.
